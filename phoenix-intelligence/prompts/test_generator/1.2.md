## Locator Priority (follow strictly)
1. `data-testid` / `data-test` attributes — most stable, always prefer when present
2. `get_by_role` — default for all interactive elements. Always provide `name=`.
3. `get_by_placeholder` — inputs without visible labels (check first before `get_by_label`)
4. `get_by_label` — ONLY when a `<label>` element is confirmed to exist
5. `get_by_text` — static text content ONLY when unique on the page. NEVER in assertions.
6. CSS selector — only when semantic locators are not possible. MUST scope to visible parent.
7. XPath — absolute last resort.

## ⛔ Strict-Mode Anti-Patterns (never generate these)

### 1. Generic selectors that match hidden elements
**WRONG** — bare `form`, `input[type='search']`, `.country-selector` match hidden elements:
```python
page.locator("form").first.fill(...)          # may match hidden localization form
page.locator("input[type='search']").fill(...)  # may match hidden country filter
page.locator(".country-selector").click()       # may match hidden disclosure wrapper
```
**CORRECT** — always scope to visible parent containers:
```python
page.locator("form[action*='contact']").fill(...)
page.locator("header input[type='search']").fill("keyword")
page.locator("button[aria-controls='country-selector']").click()
```

### 2. `get_by_label` when no `<label>` element exists
**WRONG** — OrangeHRM, many SPAs use placeholder text, not `<label>`:
```python
page.get_by_label("Username").fill("Admin")
```
**CORRECT** — use placeholder or name attribute:
```python
page.get_by_placeholder("Username").fill("Admin")
page.locator("input[name='username']").fill("Admin")
```

### 3. `get_by_label` for custom Vue/React dropdowns
**WRONG** — custom dropdowns are not native `<select>` elements:
```python
page.get_by_label("Leave Type").select_option("Annual")
```
**CORRECT** — click to open, then select option by role:
```python
page.get_by_role("combobox", name="Leave Type").click()
page.get_by_role("option", name="Annual Leave").click()
```

### 4. `get_by_text()` in assertions
**WRONG** — get_by_text() matches ALL elements containing the text:
```python
expect(page.get_by_text("Dashboard")).to_be_visible()
```
**CORRECT** — always specify the role:
```python
expect(page.get_by_role("heading", name="Dashboard")).to_be_visible()
expect(page.get_by_role("alert")).to_have_text("Successfully Saved")
```

### 5. Hardcoded user display names in locators
**WRONG** — display name changes per environment/account:
```python
page.get_by_role("button", name="Paul Collings").click()
```
**CORRECT** — target the structural element:
```python
page.locator(".oxd-userdropdown-tab").click()   # OrangeHRM
page.get_by_role("banner").get_by_role("button").last.click()
```

### 6. Clicking nested menu items without opening parent first
**WRONG** — sub-menu items are hidden until parent is expanded:
```python
page.get_by_role("link", name="Timesheets").click()
```
**CORRECT** — open parent first:
```python
page.get_by_role("link", name="Time").click()
expect(page.get_by_role("link", name="Timesheets")).to_be_visible()
page.get_by_role("link", name="Timesheets").click()
```

### 7. Strict-mode collision with exact=True
```python
page.get_by_label("Gender").select_option("Male")   # WRONG — also matches Female
page.get_by_text("Male", exact=True).check()         # CORRECT
```

## Waiting Strategy (MANDATORY)

**NEVER use:**
- `time.sleep()` — forbidden without exception
- `page.wait_for_timeout()` — forbidden unless there is a documented animation delay
- `page.wait_for_load_state("networkidle")` — broken on sites with background requests

**ALWAYS use:**
- `expect(locator).to_be_visible()` — wait for element to appear
- `page.wait_for_url(pattern)` — wait for navigation
- Element-based waits: act on the element you need next

**Navigation timeout — ALWAYS specify explicitly:**
```python
page.goto(url, timeout=120_000)  # 120s minimum for demo/staging sites
```

**After every click that changes page state:**
```python
page.get_by_role("button", name="Login").click()
page.wait_for_url("**/dashboard/**")  # or:
expect(page.locator(".dashboard-heading")).to_be_visible()
```

## Dynamic Element Patterns

### Custom dropdowns (Vue/React/Angular)
```python
# Step 1: Click trigger to open
page.get_by_role("combobox", name="Leave Type").click()
# Step 2: Wait for options
expect(page.get_by_role("listbox")).to_be_visible()
# Step 3: Select option
page.get_by_role("option", name="Annual Leave").click()
```

### Nested navigation menus
```python
# Step 1: Click parent menu to expand
page.get_by_role("link", name="Time").click()
# Step 2: Wait for sub-item to appear
expect(page.get_by_role("link", name="Timesheets")).to_be_visible()
# Step 3: Click sub-item
page.get_by_role("link", name="Timesheets").click()
```

### Date pickers
```python
# Step 1: Click the date input to open the calendar
page.locator(".oxd-date-input input").first.click()
# Step 2: Wait for calendar
expect(page.locator(".oxd-calendar-wrapper")).to_be_visible()
# Step 3: Click the date
page.get_by_text("15", exact=True).click()
```

### Shopify/disclosure patterns
```python
# Step 1: Click the trigger button (not the hidden wrapper)
page.locator("button[aria-controls='country-selector']").click()
# Step 2: Wait for options to appear
expect(page.get_by_role("option", name="United States")).to_be_visible()
# Step 3: Select option
page.get_by_role("option", name="United States").click()
```

### User dropdown / logout menus
```python
# Step 1: Open the user menu
page.locator(".oxd-userdropdown-tab").click()
# Step 2: Wait for menu item
expect(page.get_by_role("menuitem", name="Logout")).to_be_visible()
# Step 3: Click
page.get_by_role("menuitem", name="Logout").click()
```

## Assertion Patterns

### Form submission — NEVER assume role="alert"
```python
# WRONG
expect(page.get_by_role("alert")).to_be_visible()

# CORRECT — use progressive detection
original_url = page.url
page.get_by_role("button", name="Submit").click()
# Check for URL change (success = redirect)
try:
    page.wait_for_url(lambda url: url != original_url, timeout=5_000)
    # Success path — URL changed
except Exception:
    # No redirect — check for visible feedback text
    expect(
        page.locator("text=/thank|success|submitted|sent/i")
    ).to_be_visible(timeout=5_000)
```

### Element role assertions — verify actual DOM role
```python
# WRONG — "Products" in SauceDemo is a <span>, not a heading
expect(page.get_by_role("heading", name="Products")).to_be_visible()

# CORRECT — use the actual element type
expect(page.locator(".title").filter(has_text="Products")).to_be_visible()
expect(page.locator("[data-test='inventory-container']")).to_be_visible()
```

## Dialog Handling
- ALWAYS use `page.once("dialog", ...)` immediately before the action that triggers the dialog.
- NEVER use `page.on("dialog", ...)` — it stacks handlers across multiple dialogs.

```python
page.once("dialog", lambda d: d.accept())
page.get_by_role("button", name="Delete").click()
```

## Naming
- Function name: `test_<short_descriptive_name>` using snake_case (max 50 chars).

## Test Structure
- Arrange: navigate and set up state. Use `page.goto(url, timeout=120_000)`.
- Act: perform the user actions described in the acceptance criteria.
- Assert: verify expected outcomes with `expect()`.
