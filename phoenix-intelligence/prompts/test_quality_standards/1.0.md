# Test Quality Standards

> These standards are enforced in **every** generation pass. Every generated test — manual or automated — must comply with all rules below. These rules take precedence over any conflicting guidance.

---

## 1. Assertion Standards

### Assertion Types

Every test must include assertions covering the applicable types below. Do not generate tests with zero assertions.

| Type | When to use | Pattern |
|------|-------------|---------|
| **Presence** | Element is visible on screen after action | `expect(locator).to_be_visible()` |
| **Content** | Text or value matches expected outcome | `expect(locator).to_have_text(...)` / `.to_contain_text(...)` |
| **State** | Element is enabled / disabled / checked | `.to_be_enabled()` / `.to_be_checked()` |
| **Absence** | Element is gone or hidden after action | `expect(locator).to_be_hidden()` / `.not_to_be_attached()` |
| **Navigation** | URL changed as a result of an action | `expect(page).to_have_url(re.compile(r"..."))` |

### Assertion Rules

1. **Always use `expect()` — never raw `assert`.** `expect()` auto-retries until the condition is met; `assert` reads state once and fails immediately.
2. **Import `re` when using regex.** All URL pattern assertions must use `re.compile()`. The `containing=` parameter does not exist — never use it.
3. **Assert outcomes, not inputs.** After a form submit, assert the redirect or success message — do not re-read every field value.
4. **Never assume element role from visual appearance.** Derive the locator from the DOM snapshot. An element that looks like a heading may be a `<div>`. Use `get_by_text` or a stable CSS selector when the role is uncertain.

### Assertion Anti-Patterns

| Anti-Pattern | Replacement |
|---|---|
| `assert locator.is_visible()` | `expect(locator).to_be_visible()` |
| `assert page.url == "..."` | `expect(page).to_have_url(re.compile(r"..."))` |
| `assert locator.text_content() == "..."` | `expect(locator).to_have_text("...")` |
| `expect(page).to_have_url(containing="/path")` | `expect(page).to_have_url(re.compile(r".*/path.*"))` |
| `expect(page.get_by_role("alert"))` without DOM confirmation | Verify `role="alert"` exists in DOM snapshot |
| `expect(page.get_by_role("heading"))` without DOM confirmation | Verify `<h1>–<h6>` exists in DOM snapshot |

## 2. Pytest Patterns

### Test Markers

Tag **every** generated test function with at least one pytest marker:

```python
@pytest.mark.smoke         # critical happy-path; runs on every PR
@pytest.mark.regression    # full coverage; runs nightly
@pytest.mark.sanity        # quick post-deploy check
@pytest.mark.security      # negative/security scenarios
@pytest.mark.negative      # invalid inputs, error paths
@pytest.mark.boundary      # edge-case values (empty, max-length, special chars)
```

A test may carry multiple markers:

```python
@pytest.mark.regression
@pytest.mark.negative
def test_login_invalid_password(page, base_url): ...
```

### Parametrize for Multiple Scenarios

When a test covers several data variants (happy-path, boundary, error), parametrize instead of duplicating test functions:

```python
import os
import pytest

@pytest.mark.parametrize("username, password, expected_path", [
    (os.environ["TEST_USERNAME"], os.environ["TEST_PASSWORD"], r".*/dashboard"),
    ("",                          os.environ["TEST_PASSWORD"], r".*/login"),
    (os.environ["TEST_USERNAME"], "wrong_pass",                r".*/login"),
])
def test_login_scenarios(page, base_url, username, password, expected_path):
    page.goto(f"{base_url}/login", timeout=60_000)
    page.locator("input[name='username']").fill(username)
    page.locator("input[name='password']").fill(password)
    page.get_by_role("button", name="Login", exact=True).click()
    expect(page).to_have_url(re.compile(expected_path), timeout=30_000)
```

### Expected Failures and Skip

```python
@pytest.mark.xfail(reason="Bug PROJ-123: submit button incorrectly disabled")
def test_known_issue(page): ...

@pytest.mark.skip(reason="Feature not yet implemented")
def test_future_feature(page): ...
```

### expect() Catalog — Quick Reference

```python
from playwright.sync_api import expect

# Visibility / presence
expect(locator).to_be_visible()
expect(locator).to_be_hidden()
expect(locator).not_to_be_attached()

# Text
expect(locator).to_have_text("exact text")
expect(locator).to_contain_text("substring")
expect(locator).to_have_text(re.compile(r"\d+ items"))

# Value (inputs)
expect(locator).to_have_value("user@example.com")

# State
expect(locator).to_be_enabled()
expect(locator).to_be_disabled()
expect(locator).to_be_checked()
expect(locator).not_to_be_checked()
expect(locator).to_be_editable()
expect(locator).to_be_focused()

# Count
expect(locator).to_have_count(5)

# Attribute
expect(locator).to_have_attribute("href", "/docs")

# Page / URL
expect(page).to_have_url(re.compile(r".*/dashboard"))
expect(page).to_have_title(re.compile(r".*Dashboard.*"))

# Soft assertions (collect multiple failures before stopping)
expect.soft(locator).to_be_visible()
```

## 3. Waiting Standards

### Forbidden — Never Generate

| Forbidden | Reason |
|---|---|
| `time.sleep(n)` / `asyncio.sleep(n)` | No auto-retry; fixed delay; unconditionally forbidden |
| `page.wait_for_selector(...)` | Use `expect(locator).to_be_visible()` instead |
| `page.wait_for_load_state("networkidle")` | Flaky on SPAs, analytics, websockets — never reaches idle |
| `locator.wait_for(state="visible")` before `.click()` | Redundant — `.click()` auto-waits |

### Allowed

- `expect(locator).to_be_visible()` — primary wait-for-element mechanism
- `expect(locator).to_be_hidden()` / `.not_to_be_attached()` — wait for disappearance
- `expect(page).to_have_url(...)` — wait-with-assertion after navigation
- `page.wait_for_url("pattern", timeout=...)` — navigation wait before asserting
- `page.wait_for_timeout(ms)` — ONLY with an inline comment explaining the specific delay (animation or documented debounce)
- `context.expect_page()` — new tab / popup
- `page.expect_download()` — file download
- `page.expect_response(predicate)` — specific API response

### Sequencing Rule

**Register before trigger.** For browser dialogs, always call `page.once("dialog", handler)` before the action that opens the dialog:

```python
# ✅ Correct
page.once("dialog", lambda d: d.accept())
page.get_by_role("button", name="Delete").click()

# ❌ Wrong — dialog fires before handler is registered
page.get_by_role("button", name="Delete").click()
page.once("dialog", lambda d: d.accept())
```

## 4. Security and Negative Testing

### Negative Test Categories

Every module should include at least one test from each applicable category:

| Category | Test Examples |
|---|---|
| **Invalid input** | Empty required fields, values exceeding max length, special characters (`<script>`, `' OR 1=1`) |
| **Authentication** | Wrong password, locked account, blank username |
| **Authorization** | Access a resource owned by another user; access admin page as regular user |
| **Rate limiting / abuse** | Rapid repeated submissions, concurrent duplicate requests |

### test_data Fixture (Edge Cases)

When the `test_data` fixture is available (generated by TestDataEngine), use it for boundary values:

```python
def test_leave_boundary_values(page, base_url, test_data):
    edge = test_data.get("edge_cases", {})
    for days_value in edge.get("leave_days", []):
        page.goto(f"{base_url}/leave/apply", timeout=60_000)
        page.locator("input[name='days']").fill(str(days_value))
        page.get_by_role("button", name="Apply").click()
        # assert either success or validation error — not both
```

### Negative Assertion Pattern

For negative flows, always assert all three parts:

```python
# 1. Trigger the invalid action
page.get_by_role("button", name="Login").click()

# 2. Error indicator is visible
expect(page.locator("[data-testid='error-message']")).to_be_visible()

# 3. User remains on the same page (not redirected to success)
expect(page).to_have_url(re.compile(r".*/login"))
```

### Security Constraints

- **Credentials:** Always `os.environ["TEST_USERNAME"]` / `os.environ["TEST_PASSWORD"]`. Never hardcode.
- **Base URL:** Always `os.environ.get("BASE_URL")` or the `base_url` fixture. Never a hardcoded URL.
- **XSS / SQLi payloads:** Store in parameterized variables — never inline in script literals.
- **File uploads:** Always use `tempfile.NamedTemporaryFile()`. Never reference real user file paths.
- **Logging:** Never output credentials, tokens, full HTTP responses, or page HTML to logs or print statements.
- **Test accounts:** Use the lowest-privilege account that satisfies the test. Add `# Requires admin role` comment when admin credentials are legitimately needed.

## Anti-Pattern Summary

| # | Anti-Pattern | Correct Alternative |
|---|---|---|
| 1 | `time.sleep()` / `asyncio.sleep()` | `expect()` with auto-wait |
| 2 | `page.wait_for_selector(...)` | `expect(locator).to_be_visible()` |
| 3 | `page.wait_for_load_state("networkidle")` | Assert on visible content instead |
| 4 | `assert locator.is_visible()` | `expect(locator).to_be_visible()` |
| 5 | `assert page.url == "..."` | `expect(page).to_have_url(re.compile(...))` |
| 6 | `expect(page).to_have_url(containing=...)` | `re.compile(r".*/path.*")` — `containing=` does not exist |
| 7 | `get_by_role("alert")` without DOM confirmation | Check DOM snapshot for actual element type |
| 8 | `get_by_role("heading")` without DOM confirmation | Check DOM snapshot — may be a `<div>` or `<span>` |
| 9 | Hardcoded credentials in scripts | `os.environ["TEST_USERNAME"]` / `["TEST_PASSWORD"]` |
| 10 | Hardcoded base URLs | `os.environ.get("BASE_URL")` or `base_url` fixture |
| 11 | Test function missing pytest marker | At least one marker required (`smoke`, `regression`, etc.) |
| 12 | Manual step prose copied as assertion text | Assertions must be DOM-backed — never copy criterion text |
