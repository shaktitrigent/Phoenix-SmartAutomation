1. User Authentication & Session Management
Goal: Test your framework’s ability to handle secure login, invalid credentials, and session logout.

User Story: As an HR Admin, I want to log in to the system using valid credentials so that I can access the dashboard.

Acceptance Criteria:

Verify login with username Admin and password admin123.

Verify that an error message "Invalid credentials" appears for incorrect passwords.

Verify that clicking "Logout" redirects the user back to the Login page and clears the session.

Framework Testing Goal: Test Explicit Waits for the dashboard to load and Negative Testing assertions.

2. PIM: Add New Employee (Form Submission)
Goal: Test your framework's ability to interact with complex forms and toggles.

User Story: As a PIM Manager, I want to add a new employee to the system so that their record is officially registered.

Acceptance Criteria:

Navigate to PIM > Add Employee.

Input First Name, Middle Name, and Last Name.

Toggle the "Create Login Details" switch.

Upload an employee profile picture (testing file upload).

Click "Save" and verify a "Successfully Saved" toast message appears.

Framework Testing Goal: Test File Upload handling and Shadow DOM/Switch toggles.

3. Admin: System User Search (Table Interception)
Goal: Test your framework’s capability to handle dynamic web tables and filtering.

User Story: As a System Admin, I want to search for existing users by their Role and Status so that I can manage their permissions.

Acceptance Criteria:

Navigate to Admin > User Management.

Select "User Role" as ESS and "Status" as Enabled from the dropdowns.

Click "Search".

Verify that the resulting table contains only users with the ESS role.

Verify "No Records Found" logic by searching for a non-existent username.

Framework Testing Goal: Test Table scrapers, Dropdown selection, and List validations.

4. Leave: Apply for Leave (Date & Workflow)
Goal: Test logic involving date pickers and business workflows.

User Story: As an Employee, I want to apply for a "Personal Leave" for a specific date range so that my manager can approve it.

Acceptance Criteria:

Navigate to Leave > Apply.

Select "Leave Type" (e.g., CAN - Personal).

Use the Date Picker to select a "From Date" (Tomorrow) and "To Date" (Day after tomorrow).

Verify the "Partial Days" dropdown is disabled if only one day is selected.

Submit and verify the leave appears in the "My Leave" list.

Framework Testing Goal: Test JS-based Date Pickers and Calendar widgets.

5. Recruitment: Candidate Management (Dynamic UI)
Goal: Test interaction with dynamic "Add" buttons and autocomplete fields.

User Story: As a Recruiter, I want to add a new candidate and associate them with an existing vacancy.

Acceptance Criteria:

Navigate to Recruitment > Candidates.

Click the + Add button.

Enter candidate details and start typing in the "Vacancy" field to trigger the Autocomplete suggestion.

Select a vacancy from the suggestions.

Verify that the candidate status is set to "Application Initiated."

Framework Testing Goal: Test AJAX/Autocomplete fields and Dynamic Element presence.
